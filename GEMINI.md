# Project Instructions

This project follows strict engineering and design standards to ensure high-quality, performant, and accessible user interfaces.

## Design & UI Standards

All frontend development and redesign tasks MUST adhere to the **tasteskill** framework.

### Active Skills
- [tasteskill: Anti-Slop Frontend Skill](.agents/skills/taste-skill.md)

### Core Directives
1. **Brief Inference First:** Before writing any code, declare a "Design Read" (e.g., *"Reading this as: Portfolio for a developer, with a minimalist language, leaning toward Tailwind + Geist."*).
2. **Follow the Dials:** Use `DESIGN_VARIANCE`, `MOTION_INTENSITY`, and `VISUAL_DENSITY` to guide layout and animation decisions.
3. **No AI Tells:** Avoid generic gradients, centered heroes by default, and common AI boilerplate.
4. **Strict Pre-flight Check:** Always run the checklist in Section 14 of `taste-skill.md` before delivering any UI work.

## Technology Stack
- **Framework:** Next.js (App Router)
- **Styling:** Tailwind CSS
- **Animation:** Framer Motion (use `motion/react`)
- **Icons:** Phosphor Icons / Radix Icons
